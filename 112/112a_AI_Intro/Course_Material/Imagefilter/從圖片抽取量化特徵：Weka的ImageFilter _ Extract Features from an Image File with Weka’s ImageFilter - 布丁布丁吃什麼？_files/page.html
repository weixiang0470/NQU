要讓網頁應用程式變成更具生產力的工具，多螢幕的支援是不可或缺的關鍵。

https://web.dev/multi-screen-window-placement/

即將在3月底迎來的Chrome 100加入了Multi-Screen Window Placement API。⋯⋯
它的作用是讓網頁偵測到使用者現在的延伸桌面(extended desktop)、判斷主副視窗，並讓視窗擺到對應的螢幕上。

Thomas Steiner舉了幾個其他應用程式的常見案例：
- GIMP之類的圖片編輯器會同時使用多個視窗(工具) (這樣說起來，剪影片的時候大家也蠻常會用多螢幕延伸桌面)。
- 用多個視窗顯示股票變化，各個視窗可以放大全螢幕檢視。
- 投影片應用程式中，主講者可以把講稿顯示在自己的螢幕，而投影片放到投影機上。

這真是我這種多螢幕使用者的福音啊！

Linux桌面環境KDE裡有個令人困惑的設定：活動(Activity)。

前些日子我們有提到KDE最新版Overview功能，可以讓人方便在多個「虛擬桌面」(Virtual Desktop)搬移應用程式視窗。
很多作業系統都有「虛擬桌面」，那「活動」到底是什麼呢？

⋯⋯

https://chakra-zh.blogspot.com/2012/06/kde-activity.html

簡單來說，「活動」可以算是「虛擬桌面」的加強版。不同「活動」的桌面可以有所差異，包括桌布和小工具(Widget)，但不同「虛擬桌面」都使用同樣設定，除非是不同顯示器。

不同「活動」的應用程式使用記錄也可以有所區別。
例如我在名為「論文」的活動中使用較多Zotero跟PDF閱讀器，該活動就會把它記錄在最常使用的應用程式中。
在另一個名為「遊戲」的活動可能較常開啟Steam，而且我也可以為該活動指定電源管理，調整螢幕關閉的時間。

https://askubuntu.com/a/399394
除此之外，「活動」可以客製化的狀態還有語言、權限、電子郵件、日曆等等。

不過目前最新的Overview功能只能在「虛擬桌面」切換，而且活動切換器(Activity Pager)操作起來也不太方便。不知道KDE之後會不會繼續更新「活動」功能呢.....？

星期四

你有自己試過把書籍掃描數位化成為PDF過嗎？

現在很多掃描工具都內建了轉換成PDF功能，方便我們把很多原本是圖片的掃描檔結合成一個PDF。
PDF檔案也讓我們方便進一步做文字識別。

⋯⋯

然而，高畫質的圖片會使得PDF檔案非常龐大，這時候該怎麼辦呢？
如果是使用Windows的Acrobat Pro，我就可以用PDF最佳化程式降低儲存解析度，有效降低檔案大小。

那在Linux裡，我們要怎麽壓縮PDF的檔案大小呢？

http://www.alfredklomp.com/programming/shrinkpdf/

一般做法是使用Ghostscript指令gs來壓縮PDF，而Alfred Klomp將Ghostscript指令整理成shrinkpdf.sh
它預設將PDF裡的圖片解析度降低為72 DPI，以此壓縮PDF的檔案。

我最近拿了一個250MB的PDF檔案來壓縮看看，結果出來的檔案只剩下28MB。
當然，PDF圖片畫質的確沒有原始檔案的好，但很多時候印出來感覺也沒差多少。

總而言之，沒想到Linux連壓縮PDF的方法也有，真的是太神啦！
